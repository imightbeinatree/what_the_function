 12   # = ERb
 13   # 
 14   # You trigger ERb by using embeddings such as <% %>, <% -%>, and <%= %>. The <%= %> tag set is used when you want output. Consider the 
 15   # following loop for names:
 16   #
 17   #   <b>Names of all the people</b>
 18   #   <% for person in @people %>
 19   #     Name: <%= person.name %><br/>
 20   #   <% end %>
 21   #
 22   # The loop is setup in regular embedding tags <% %> and the name is written using the output embedding tag <%= %>. Note that this
 23   # is not just a usage suggestion. Regular output functions like print or puts won't work with ERb templates. So this would be wrong:
 24   #
 25   #   Hi, Mr. <% puts "Frodo" %>
 26   #
 27   # If you absolutely must write from within a function, you can use the TextHelper#concat.
 28   #
 29   # <%- and -%> suppress leading and trailing whitespace, including the trailing newline, and can be used interchangeably with <% and %>.
 30   #
 31   # == Using sub templates
 32   #
 33   # Using sub templates allows you to sidestep tedious replication and extract common display structures in shared templates. The
 34   # classic example is the use of a header and footer (even though the Action Pack-way would be to use Layouts):
 35   #
 36   #   <%= render "shared/header" %>
 37   #   Something really specific and terrific
 38   #   <%= render "shared/footer" %>
 39   #
 40   # As you see, we use the output embeddings for the render methods. The render call itself will just return a string holding the
 41   # result of the rendering. The output embedding writes it to the current template.
 42   #
 43   # But you don't have to restrict yourself to static includes. Templates can share variables amongst themselves by using instance
 44   # variables defined using the regular embedding tags. Like this:
 45   #
 46   #   <% @page_title = "A Wonderful Hello" %>
 47   #   <%= render "shared/header" %>
 48   #
 49   # Now the header can pick up on the <tt>@page_title</tt> variable and use it for outputting a title tag:
 50   #
 51   #   <title><%= @page_title %></title>
 52   #
 53   # == Passing local variables to sub templates
 54   # 
 55   # You can pass local variables to sub templates by using a hash with the variable names as keys and the objects as values:
 56   #
 57   #   <%= render "shared/header", { :headline => "Welcome", :person => person } %>
 58   #
 59   # These can now be accessed in <tt>shared/header</tt> with:
 60   #
 61   #   Headline: <%= headline %>
 62   #   First name: <%= person.first_name %>
 63   #
 64   # If you need to find out whether a certain local variable has been assigned a value in a particular render call,
 65   # you need to use the following pattern:
 66   #
 67   #   <% if local_assigns.has_key? :headline %>
 68   #     Headline: <%= headline %>
 69   #   <% end %>
 70   #
 71   # Testing using <tt>defined? headline</tt> will not work. This is an implementation restriction.

254       if options.is_a?(String)

267           else

294     # Returns a symbolized version of the <tt>:format</tt> parameter of the request,
295     # or <tt>:html</tt> by default.
296     #
297     # EXCEPTION: If the <tt>:format</tt> parameter is not set, the Accept header will be examined for
298     # whether it contains the JavaScript mime type as its first priority. If that's the case,
299     # it will be used. This ensures that Ajax applications can use the same URL to support both
300     # JavaScript and non-JavaScript users.
301     def template_format
302       return @template_format if @template_format
303 
304       if controller && controller.respond_to?(:request)
305         parameter_format = controller.request.parameters[:format]
306         accept_format    = controller.request.accepts.first
307 
308         case
309         when parameter_format.blank? && accept_format != :js
310           @template_format = :html
311         when parameter_format.blank? && accept_format == :js
312           @template_format = :js
313         else
314           @template_format = parameter_format.to_sym
315         end
316       else
317         @template_format = :html
318       end
319     end
